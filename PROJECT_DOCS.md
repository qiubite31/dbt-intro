# dbt Project Overview

This document provides a comprehensive guide to the `dbt-intro` project, which uses dbt (data build tool) with SQLite to model customer and order data.

## Project Structure

The project follows a standard dbt directory structure:

```
dbt-intro/
├── dbt_project.yml         # Main project configuration file
├── profiles.yml            # Database connection profile configuration
├── data.db                 # SQLite database file (created by setup_data.py)
├── setup_data.py           # Python script to generate sample data and create the database
├── run_dbt_tasks.py        # Python script to automate dbt run, test, and docs generate
├── query_data.py           # Python script to verify and query data in SQLite
├── drop_dbt_model.py       # Python script to clean up dbt models from SQLite
├── models/                 # Contains SQL model files
│   ├── schema.yml          # Schema definitions and tests for models
│   ├── stg_customers.sql   # Staging model for customers
│   ├── stg_orders.sql      # Staging model for orders
│   ├── customer_order_detail.sql # Detailed order view (materialized as table)
│   └── customers_with_orders.sql  # Final model joining customers and orders
├── logs/                   # dbt log files
└── target/                 # Compiled SQL files and run artifacts (generated by dbt)
```

## Key Files Description

### Configuration

*   **`dbt_project.yml`**:
    *   **Project Name**: `my_sqlite_project`
    *   **Profile**: `my_sqlite_profile`
    *   **Model Paths**: `models`
    *   **Default Materialization**: `view` (for models in `my_sqlite_project`)

*   **`profiles.yml`**:
    *   Defines the `my_sqlite_profile` configuration.
    *   Connects to the `data.db` SQLite database located in the project root.
    *   Target schema: `main`

*   **`setup_data.py`**:
    *   A Python script that initializes the SQLite database `data.db`.
    *   Creates two raw tables: `raw_customers` and `raw_orders` with sample data.

### Models (`models/`)

The models directory contains the transformation logic written in SQL with Jinja templating.

1.  **`stg_customers.sql`**:
    *   **Purpose**: Cleans and renames raw customer data.
    *   **Source**: `raw_customers` table.
    *   **Transformations**: Renames `id` to `customer_id`.

2.  **`stg_orders.sql`**:
    *   **Purpose**: Cleans and renames raw order data.
    *   **Source**: `raw_orders` table.
    *   **Transformations**: Renames `id` to `order_id` and `user_id` to `customer_id`.

3.  **`customers_with_orders.sql`**:
    *   **Purpose**: Creates a rich customer view including order statistics.
    *   **Dependencies**: `stg_customers`, `stg_orders`.
    *   **Logic**:
        *   Aggregates orders by `customer_id` to calculate:
            *   `first_order_date`
            *   `most_recent_order_date`
            *   `number_of_orders`
        *   Joins these metrics back to the customer details.
        *   Handles customers with no orders (coalescing order count to 0).

4.  **`customer_order_detail.sql`**:
    *   **Purpose**: Joins customers and orders to provide a detailed view of each order.
    *   **Dependencies**: `stg_customers`, `stg_orders`.
    *   **Materialization**: `table`.

5.  **`schema.yml`**:
    *   Defines the sources (`raw_customers`, `raw_orders`) and models.
    *   Configures tests (e.g., unique, not null) and documentation for columns.

## Getting Started

1.  **Initialize Data**: Run `python setup_data.py` to create a fresh `data.db`.
2.  **Run dbt Automation**: Execute `python run_dbt_tasks.py` to compile, run tests, and generate documentation.
3.  **Verify Data**: Execute `python query_data.py` to verify the results in SQLite.
4.  **Clean up (Optional)**: Execute `python drop_dbt_model.py` to remove dbt-generated models from SQLite.
